# Stage 4B Proof Outputs

These plain-text captures are the source for Google Drive screenshots. They are
generated by:

```bash
bash scripts/capture_evidence.sh
```

| Stage 4B proof | File | What it proves |
|---|---|---|
| validate output | `01_validate.txt` | Image exists, port is free, generated nginx config passes syntax check. |
| hard pre-deploy gate | `02_predeploy_policy_denial.txt` | Raising the disk threshold makes OPA deny deploy before app/nginx start. |
| deploy and metrics | `03_deploy_and_metrics.txt` | OPA starts first, infrastructure policy passes, app/nginx become healthy, `/metrics` exposes Prometheus text. |
| status history | `04_status_history.txt` | `status --once` calculates metrics, shows policy compliance, and appends `history.jsonl`. |
| promote canary | `05_promote_canary.txt` | Promote is policy-gated and canary is confirmed by body mode plus `X-Mode`. |
| promote denied under chaos | `06_promote_denied_under_chaos.txt` | Error chaos causes canary safety policy denial before manifest mutation. |
| promote stable | `07_promote_stable.txt` | Recovery followed by policy-gated stable promotion. |
| OPA no leakage | `08_opa_no_leakage.txt` | OPA is reachable on localhost for the CLI but not through the Nginx port. |
| audit report | `09_audit_report.txt` | `audit_report.md` is generated from `history.jsonl` and lists policy violations. |
| generated configs | `10_generated_configs.txt` | Manifest, SHA256 values, full generated Compose and Nginx configs. |
| teardown and regen | `11_teardown_and_regen.txt` | `teardown --clean` removes generated configs; `init` recreates them. |

The image tag must match `manifest.yaml` (`services.image`), currently
`swiftdeploy-stage4b-app:1.0.0`.
