# 2026-05-03 - SD4-APP-001 closed

## What I built

`app/main.py` (FastAPI), `app/requirements.txt`, `app/__init__.py`, plus a
reusable smoke harness at `scripts/smoke_app.py`. Three routes (`/`, `/healthz`,
`POST /chaos`) and two middlewares (chaos effects, X-Mode header). 26
assertions executed in stable and canary modes via FastAPI TestClient, plus a
live uvicorn server probed with curl over a TCP socket.

## What I can defend without AI

- **`time.monotonic()` for uptime**: it only moves forward, so NTP slews and
  manual `date` changes cannot make uptime decrease.
- **X-Mode is middleware, not per-route**: the brief says "every response,"
  which includes 4xx and 5xx. The X-Mode middleware is registered second on
  purpose so it is the *outer* wrapper and sees the chaos-injected 500. Smoke
  test #23 directly asserts `X-Mode: canary` on a chaos 500 response.
- **`/chaos` is exempt from chaos effects**: recovery escape hatch. The smoke
  proves error rate 1.0 still allows `POST /chaos {recover}` to succeed.
- **Stable-mode `/chaos` returns 403 *before* Pydantic parses the body**:
  policy answer first; body shape is irrelevant when the route is disabled.
- **`Literal["slow","error","recover"]`**: the closed set is enforced by the
  type system rather than handler conditionals.
- **Why the chaos middleware returns `JSONResponse` instead of raising**:
  direct response is one trip through the stack and a predictable header path.

## What I tested and what the numbers said

- Stable `/`: no `X-Mode`, `mode=stable` in body. PASS.
- Canary `/`: `x-mode: canary` header, `mode=canary` in body. PASS.
- Chaos slow 0.4s: `/healthz` measured at 0.418s (in-process) and 1.078s for
  duration=1s (over uvicorn). The server is genuinely sleeping. PASS.
- Chaos error 1.0: every non-`/chaos` route returns 500. `POST /chaos
  {recover}` still 200, and `/healthz` returns to fast happy-path after
  recover. PASS.
- Invalid chaos mode: returns 400. PASS.

## Risks I noticed and parked

- `uvicorn[standard]` pulls `uvloop` which is glibc-only. That is fine for
  `python:3.11-slim` (Debian/glibc). It would break on alpine — but D-004
  already locked us to slim, so no exposure.
- `chaos_state` is module-global. Multiple uvicorn workers would each have
  their own state. Compose runs a single container, so single worker, so this
  is moot — but worth flagging during interview if asked about scaling.
