## Bugs caught (and the lesson each one teaches)

1. **Microsoft Store stub for `python3`**: `command -v python3` resolved a
   non-functional Windows shim. Fixed by probing each candidate with
   `--version` before accepting it. Lesson: PATH lookups are necessary but
   not sufficient — probe before you trust.

2. **Heredoc path embedding**: check 1 inlined `${MANIFEST}` directly into a
   Python heredoc, sending the Git Bash path `/c/Users/...` to a Windows
   Python that cannot resolve it. Fixed by passing the path via `argv` (where
   MSYS auto-translates the path) and using a quoted heredoc delimiter
   `'PYEOF'` so the shell does not expand inside the body. The other 3 Python
   blocks already used this pattern; check 1 was the lonely odd one out.

## What I can defend without AI

- **Why atomic-write (temp file + `os.replace`)**: a partial render mid-crash
  must not leave an unparseable `nginx.conf` at the canonical path.
- **Why the 5 checks are ordered the way they are**: each one assumes the
  previous one passed. Cheapest upstream check first.
- **Why `socket.bind()` instead of `ss -tln`**: same kernel call nginx will
  make. Race-free; cross-platform; stdlib only.
- **Why `--entrypoint nginx -t -q` for syntax check**: bypasses the upstream
  image's setup scripts that would otherwise mutate the config and dirty the
  output.
